## 产品定义

上传代码项目,自动生成依赖图谱,通过**三层可视化视图**和AI辅助,安全地完成代码重构。

**核心价值**：
> "在IDE中需要一个一个找函数才知道调用关系,在CodeFlow中可以直接看到跨文件的函数调用链"

---

## 核心概念

### 三层视图体系（全新设计）

| 视图层级 | 用途 | 节点类型 | 布局方式 |
|---------|------|---------|---------|
| **📦 模块视图** | 文件间依赖 | 文件/模块 | 力导向图 |
| **🏛️ 类视图** | 类结构关系 | 类/独立函数 | 继承树布局 |
| **⚙️ 函数视图** | 跨文件调用链 | 函数/方法 | **泳道式布局** |

**关键变化**：
- ❌ 不再是混合节点展开/折叠
- ✅ 三层完全独立,通过Tab或钻取切换
- ✅ 函数视图是核心,采用创新的泳道布局

### 视图导航模式

```
全局切换：[📦 模块] [🏛️ 类] [⚙️ 函数] ← 顶部Tab

渐进钻取：
  模块视图: 点击文件 → 进入该文件的类视图
  类视图:   点击类   → 进入该类的函数视图
  函数视图: 点击函数 → 高亮调用链 + 显示详情

面包屑：CodeFlow / 📦 所有模块 / 🏛️ api_client.py / ⚙️ get_status()
```

### 节点状态（4种）

| 状态 | 含义 | 视觉 |
|------|------|------|
| 正常 | 无修改 | 默认样式 |
| 已修改 | 待保存 | 橙色边框 + 闪烁动画 |
| 异步冲突 | sync调async | 红色警告图标 ⚠️ |
| 类型不匹配 | schema不兼容 | 黄色警告图标 ⚠️ |

### 连接线类型

| 关系 | 线条样式 | 颜色 | 箭头 | 使用场景 |
|------|----------|------|------|---------|
| 函数调用 | 实线 | `#4A90E2` (蓝) | → | 函数视图 |
| 继承关系 | 虚线 | `#7B68EE` (紫) | ⇢ | 类视图 |
| 引用/导入 | 点线 | `#50C878` (绿) | ↔ | 模块视图 |
| 待保存修改 | 加粗实线 | `#FF6B6B` (橙红) | ⟹ | 所有视图 |
| 跨泳道调用 | 虚线+节点 | `#4A90E2` (蓝) | •N→ | 函数视图(折叠) |

### 工作模式

```
浏览模式 → 编辑模式 → 分析模式 → 应用模式
    ↑___________________________________|
```

- **浏览模式**：在三层视图间导航,查看依赖关系
- **编辑模式**：拖拽创建/删除调用关系（主要在函数视图）
- **分析模式**：AI分析影响链,显示修复建议
- **应用模式**：预览Diff,应用修改到代码

## 核心工作流

### 工作流1：探索代码结构

```
1. 进入模块视图
   ↓ 看到项目有3个主要模块
   
2. 点击 api_client.py
   ↓ 进入该文件的类视图
   
3. 看到 APIClient 类和几个独立函数
   ↓ 点击 APIClient 类
   
4. 进入函数视图,看到该类的所有方法
   ↓ 点击 get_status() 函数
   
5. 高亮显示完整的调用链
   ↓ 右侧面板显示详细信息
   
6. 发现问题：异步调用冲突
```

### 工作流2：修复异步冲突（在函数视图中）

```
1. 在函数视图中,点击 get_status()
   ↓ 看到它被 process_batch() 调用
   ↓ 右侧面板提示：异步冲突
   
2. 点击"修复建议"
   ↓ 弹出影响链分析
   
   ┌──────────────────────────────┐
   │ 🔼 影响链分析                 │
   ├──────────────────────────────┤
   │                              │
   │ Layer 1: get_status()        │
   │ ✓ 改为 async def             │
   │ [已应用]                      │
   │                              │
   │ Layer 2: process_batch() 🟢  │
   │ • 修改: def → async def      │
   │ • AI分析: 安全修复           │
   │ [🔧 包含在自动修复]           │
   │                              │
   │ Layer 3: api_endpoint() 🔴   │
   │ • 修改: def → async def      │
   │ • 风险: FastAPI路由入口      │
   │ [⚠️ 需要确认]                │
   │                              │
   │ [应用Layer 1-2] [全部应用]   │
   │                              │
   └──────────────────────────────┘
   
3. 用户选择"应用Layer 1-2"
   ↓ 后端生成代码修改
   
4. 显示 Diff 预览
   ↓ 用户确认
   
5. 应用修改,依赖图自动刷新
   ↓ 泳道中的节点状态更新
   ↓ 异步冲突标记消失
```

### 工作流3：拖拽创建新调用关系（在函数视图中）

```
1. 用户想让 validate() 调用新的 check_format()
   
2. 切换到"编辑模式"
   
3. 在函数视图中:
   - 找到 validator.py 泳道
   - 找到 validate() 函数节点
   
4. 拖拽 validate() 的输出端点
   ↓ 出现虚线预览
   
5. 拖到 utils.py 泳道的 check_format() 节点
   ↓ 如果目标节点兼容:高亮显示
   ↓ 如果不兼容:显示禁止图标
   
6. 释放鼠标
   ↓ 创建蓝色实线连接
   ↓ 节点状态变为"已修改"（橙色边框）
   
7. AI自动分析:
   "需要在 validate() 中添加对 check_format() 的调用"
   
8. 显示建议的代码修改
   ↓ 用户确认后应用
```

## 特殊交互说明

### 泳道动态折叠逻辑

**触发时机**：
- 用户点击某个函数节点
- 用户点击调用链中的某个函数

**执行步骤**：
1. 分析该函数的调用链（上游+下游）
2. 提取调用链涉及的所有文件
3. 展开这些文件的泳道
4. 折叠其他无关文件的泳道
5. 添加展开/折叠动画（300ms）

**示例**：
```
点击前:
╔════╗ ╔════╗ ╔════╗ ╔════╗
║file1║ ║file2║ ║file3║ ║file4║
╚════╝ ╚════╝ ╚════╝ ╚════╝
全部展开

点击 file2 中的某个函数（它调用 file4 的函数）:

╔════╗ ╔════╗ ╔═╗ ╔════╗
║file1║ ║file2║ ║3║ ║file4║
╚════╝ ╚════╝ ╚═╝ ╚════╝
折叠(灰) 展开    折叠 展开

file3 折叠（因为不在调用链中）
file1 折叠（因为不在调用链中）
```

### 视图钻取面包屑

**生成规则**：
```typescript
function generateBreadcrumb(view: ViewType, scope?: string): string {
  const parts = ['CodeFlow'];
  
  if (view === 'module') {
    parts.push('📦 所有模块');
  }
  
  if (view === 'class' && scope) {
    parts.push('📦 所有模块', `🏛️ ${scope}`);
  }
  
  if (view === 'function' && scope) {
    const [file, className] = scope.split('::');
    parts.push('📦 所有模块', `🏛️ ${file}`);
    if (className) {
      parts.push(`⚙️ ${className}`);
    } else {
      parts.push('⚙️ 函数视图');
    }
  }
  
  return parts.join(' / ');
}
```

**示例**：
```
模块视图: CodeFlow / 📦 所有模块
类视图:   CodeFlow / 📦 所有模块 / 🏛️ api_client.py
函数视图: CodeFlow / 📦 所有模块 / 🏛️ api_client.py / ⚙️ APIClient
```
