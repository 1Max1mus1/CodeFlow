# 📘 CodeFlow - 三层分离视图体系设计（V3更新）

## 本次更新要点
基于用户反馈,重新设计节点层级体系:
- ✅ **三层视图完全分离** - 不再是混合在一起的节点展开/折叠
- ✅ **函数级视图采用泳道式布局** - 清晰展示跨文件调用关系
- ✅ **动态折叠机制** - 只展开相关文件,避免视觉混乱
- ✅ **核心价值聚焦** - 强化"跨文件函数调用可视化"的产品定位

---

## 3.2 三层分离的视图体系设计

**核心理念**: 三层视图完全独立,专注展示不同粒度的调用关系。用户通过全局切换或点击节点在视图间导航。

---

### 3.2.1 视图1: 模块级视图（📦 Module View）

**用途**: 展示项目的文件间依赖关系,适合理解整体架构

**布局方式**: 
- 节点：每个文件/模块一个节点
- 连接线：import/from导入关系
- 排列：自动布局（力导向图或分层布局）

**节点设计**:
```
┌─────────────────────────────────┐
│ 📦 api_client.py                │
│                                 │
│ 📊 3 classes | 12 functions     │
│ 📥 imports: requests, json      │
│ 📤 used by: processor.py        │
│                                 │
│ [🔍 钻取到类视图]                │ ← 点击进入该文件的类视图
└─────────────────────────────────┘
```

**交互**:
- 点击节点 → 进入该文件的类视图
- 悬浮显示 → 文件统计信息、主要导出内容
- 右侧面板 → 显示模块的README、主要功能说明

**示例场景**:
```
┌──────────────┐     imports     ┌──────────────┐
│ api_client.py├────────────────→│ utils.py     │
└──────┬───────┘                 └──────────────┘
       │ imports
       ↓
┌──────────────┐
│processor.py  │
└──────────────┘
```

**用途**: 展示文件内的类结构和类间关系（继承、组合等）

**布局方式**: 
- 节点：每个类/独立函数一个节点
- 连接线：继承、Mixin、类间调用
- 排列：继承树布局（父类在上）

**节点设计**:
```
┌────────────────────────────────┐

│                                │

**交互**:
- 点击节点 → 进入该类的函数视图
- 悬浮显示 → 类的方法列表、属性列表

**示例场景**:

### 3.2.3 视图3: 函数级视图（⚙️ Function View） - **核心视图**

**用途**: 展示跨文件的函数调用关系链,这是产品的核心价值所在

**核心价值主张**:
> "在 IDE 中需要一个一个找函数才知道调用关系,在 CodeFlow 中可以直接看到跨文件的函数调用链"

**布局方式**: **垂直泳道式（Swimlane）**
- 每个文件是一个垂直泳道（从左到右排列）
- 调用关系的箭头从左往右流动
- 动态折叠：只展开当前关注的调用链涉及的文件

**泳道设计**:

**展开状态的泳道**:
```

║ 📄 api_client.py          ║
╠═══════════════════════════╣

║  │ ⚙️ get()            │  ║
║  │ Line 45-67          │  ║
║  │ Returns: Response   │  ║
║  │ Async: ⚡ Yes       │  ║
║  └─────────────────────┘  ║
║           │               ║
║           │ calls         ║
║  ┌────────▼────────────┐  ║
║  │ ⚙️ post()           │  ║
║  │ Line 89-102         │  ║
║  └─────────────────────┘  ║
║                           ║

**交互行为（函数视图 - 最重要）**:

**点击函数节点时触发三件事**:

1. **高亮调用链**: 
   - 上游调用者和下游被调用者的函数节点高亮
   - 相关箭头变粗+高亮颜色（如橙色）
   
2. **自动展开相关泳道**:
   - 调用链路径上的所有文件泳道自动展开
   - 无关文件的泳道自动折叠成窄边栏
   
3. **右侧详情面板显示**:
   - 函数代码片段（语法高亮）
   - 参数和返回值详细说明
   - 调用链路径列表（可点击跳转）
   - AI分析："这个函数的作用是..."

## 设计优势总结

### 相比原设计的改进

**原设计（混合节点）**:
```
❌ 问题:
- 模块、类、函数混在一个画布上
- function 直接连到 file/class,视觉混乱
- 无法清晰看到跨文件调用关系
```

**新设计（三层分离）**:
```
✅ 优势:
- 三层完全独立,每层专注一个关注点
- 函数视图采用泳道式,清晰展示文件边界
- 跨文件调用一目了然（箭头跨越泳道）
- 动态折叠避免信息过载
```

### 核心场景对比

**场景**: 开发者想知道 `get_status()` 调用了哪些其他文件的函数

**在 IDE 中**:
1. 打开 `api_client.py`
2. 找到 `get_status()` 函数
3. 看到它调用了 `client.get()`
4. Cmd+Click 跳转到 `client.py`
5. 看到 `get()` 又调用了 `parse_json()`
6. Cmd+Click 跳转到 `processor.py`
7. 重复N次...最后可能忘了最开始在哪

**在 CodeFlow 中**:
1. 打开函数视图
2. 点击 `get_status()` 节点
3. ✅ **直接看到完整的调用链,跨越3个文件**
4. ✅ **箭头清晰显示调用方向**
5. ✅ **右侧面板显示每个函数的详情**
6. ✅ **可以直接从链条上点击任意函数查看**

## 技术实现要点

### 泳道布局算法

```typescript
interface Swimlane {
  id: string;              // 文件路径
  collapsed: boolean;      // 是否折叠
  width: number;           // 展开时宽度
  collapsedWidth: number;  // 折叠时宽度(60px)
  functions: FunctionNode[];
}

// 动态折叠逻辑
function updateSwimlanes(selectedNode: FunctionNode) {
  // 1. 找出所有相关的文件
  const relatedFiles = getCallChain(selectedNode)
    .map(node => node.file);
  
  // 2. 展开相关文件,折叠其他
  swimlanes.forEach(lane => {
    lane.collapsed = !relatedFiles.includes(lane.id);
  });
  
  // 3. 重新布局
  layoutSwimlanes();
}
```

### 跨泳道箭头渲染

```typescript
// 检测箭头是否穿过折叠的泳道
function renderEdge(source: FunctionNode, target: FunctionNode) {
  const collapsedBetween = swimlanes
    .filter(lane => lane.collapsed)
    .filter(lane => isBetween(lane, source.file, target.file));
  
  if (collapsedBetween.length > 0) {
    // 渲染虚线+中间节点
    return (
      <DashedEdge 
        midpoint={{
          label: `•${collapsedBetween.length}`,
          tooltip: `途经: ${collapsedBetween.map(l => l.id).join(', ')}`
        }}
      />
    );
  } else {
    // 渲染普通实线
    return <SolidEdge />;
  }
}
```
